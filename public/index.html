How it plugs in

SwapIQ is a stateless service between systems the retailer already runs. This page is a live client of these exact endpoints.

// picker app emits a stockout POST /api/swap-offer { "order_id": "ORD-88213", "out_of_stock_sku": "P001", "shopper_id": "S1" } 200 OK { "best": { "name": "FreshFarm Oat Milk Barista", "reason": "Nut-free vegan plant milk, within Rs.2 of your usual.", "safety": "proven by graph traversal" }, "safe_count": 18, "blocked_count": 5, "graph_ms": 1.2, "rank_s": 0.9 } // customer taps in their app POST /api/decide { "accepted": true }

What the retailer provides

  • Product catalog with ingredient data (FSSAI labels already carry it)
  • Shopper dietary profiles, one screen at signup
  • The stockout event from the picker app
  • The accept or refund tap, one webhook back

Scaling path

  • This demo: in-memory graph, 200 SKUs, instant
  • Production: identical schema on Neo4j, millions of SKUs
  • One LLM call per stockout, about one second
  • Stateless, horizontally scalable, no PII stored
