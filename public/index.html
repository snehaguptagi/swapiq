How SwapIQ plugs into QuickCart

A stateless service between systems the retailer already runs.

// picker app emits a stockout POST /api/swap-offer { "order_id": "QC-88213", "out_of_stock_sku": "P001", "shopper_id": "S1" } 200 OK { "best": { "name": "FreshFarm Oat Milk Barista", "reason": "Nut-free vegan plant milk, within Rs.2 of your usual.", "safety": "proven by graph traversal" }, "safe_count": 18, "blocked_count": 5 }

What the retailer provides

  • Product catalog with ingredient data (FSSAI labels already carry it)
  • Shopper dietary profiles, one signup screen
  • The stockout event from the picker app
  • The accept or refund tap, one webhook

Scaling path

  • This demo: in-memory graph, 500+ SKUs, 24 categories
  • Production: same schema on Neo4j, millions of SKUs
  • One LLM call per stockout, about one second
  • Stateless, horizontally scalable
